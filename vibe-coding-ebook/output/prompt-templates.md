## Part 1. 바이브 코딩 시작하기

### 템플릿 01. 사업 아이디어 발견하기 (Chapter 3)

```
나는 [나이]대 [직업/경력]이야.
[관심 분야/취미]에 관심이 많아.

매일 겪는 불편함 중에서 사업 아이디어가 될 수 있는 것을
5가지 제안해줘.

조건:
- 혼자서도 만들 수 있는 규모
- 코딩 경험이 없어도 AI로 만들 수 있는 것
- 월 10~50만원 수익이 가능한 것
```

### 템플릿 02. 아이디어 구체화하기 (Chapter 3)

```
나는 [아이디어 한 줄 설명] 서비스를 만들고 싶어.

이 아이디어에 대해 아래 내용을 정리해줘:
1. 이 서비스가 필요한 사람은 누구인지
2. 비슷한 서비스가 이미 있는지
3. 내 서비스만의 차별점은 뭘 수 있는지
4. 가장 먼저 만들어야 할 핵심 기능 3가지
5. 예상 수익 모델

쉬운 말로 설명해줘.
```

## Part 2. 첫 걸음 내딛기

### 템플릿 03. AI와 첫 대화 나누기 (Chapter 4)

```
안녕, 나는 코딩을 전혀 모르는 [나이]대야.
바이브 코딩으로 웹서비스를 만들어보고 싶어.

먼저 내가 알아야 할 기본 개념을
초등학생도 이해할 수 있게 설명해줘.

특히 이런 용어들을 쉽게 알려줘:
- HTML, CSS, JavaScript
- 서버와 클라이언트
- 배포(deploy)
```

### 템플릿 04. 자기소개 웹페이지 만들기 (Chapter 5)

```
나의 자기소개 웹페이지를 만들어줘.

포함할 내용:
- 이름: [이름]
- 한 줄 소개: [소개]
- 취미 3가지: [취미1], [취미2], [취미3]
- 좋아하는 명언: [명언]
- 연락처: [이메일]

디자인:
- 색상은 [따뜻한/차가운/밝은] 느낌으로
- 글씨는 크고 읽기 쉽게
- 사진 넣을 공간 포함
- 모바일에서도 잘 보이게
```

### 템플릿 05. 웹페이지 수정 요청하기 (Chapter 5)

```
방금 만든 웹페이지에서 이것들을 수정해줘:

1. 배경색을 [색상]으로 바꿔줘
2. 제목 글씨를 더 [크게/작게] 해줘
3. [요소]를 [위치]에 추가해줘
4. 전체적으로 [느낌] 분위기로 바꿔줘

나머지는 그대로 유지해줘.
```

### 템플릿 06. 배포 도움 요청 (Chapter 6)

```
내가 만든 웹페이지를 Netlify에 배포하고 싶어.

지금 내 파일 구조는 이래:
- index.html
- style.css
- [기타 파일]

초보자도 따라할 수 있게
Netlify 배포 과정을 단계별로 알려줘.
스크린샷 설명처럼 자세하게 부탁해.
```

## Part 3. 실전 프로젝트

### 템플릿 07. 프로젝트 기획하기 (Chapter 7)

```
나는 [아이디어] 서비스를 만들려고 해.

아래 형식으로 1페이지 기획서를 작성해줘:

1. 서비스 한 줄 설명
2. 해결하려는 문제
3. 타겟 사용자
4. 핵심 기능 3가지 (MVP)
5. 나중에 추가할 기능 (우선순위 낮음)
6. 경쟁 서비스와의 차별점

주의: 처음부터 완벽할 필요 없어.
60점짜리 MVP로 시작하는 것에 초점을 맞춰줘.
```

### 템플릿 08. 기능 추가 요청하기 (Chapter 8)

```
지금 만들고 있는 [서비스명]에
[기능 이름] 기능을 추가해줘.

구체적으로:
- [기능의 동작 설명]
- [사용자가 어떤 버튼을 누르면 어떤 일이 일어나는지]
- [결과가 어떻게 보여야 하는지]

다른 기능은 건드리지 말고,
이 기능만 추가해줘.
```

### 템플릿 09. 에러 해결 요청하기 (Chapter 8)

```
아래 에러가 나왔어.
원인을 알려주고 고쳐줘.

[에러 메시지를 여기에 붙여넣기]

상황 설명:
- [무엇을 하려고 했는지]
- [언제부터 에러가 나는지]
- [어떤 것을 바꿨는지]

초보자도 이해할 수 있게 원인을 설명해주고,
수정된 코드를 보여줘.
```

### 템플릿 10. 디자인 다듬기 (Chapter 9)

```
지금 내 웹페이지의 디자인을 개선해줘.

원하는 느낌:
- 전체 색상: [파란색/초록색/따뜻한 색] 계열
- 분위기: [깔끔한/따뜻한/전문적인]
- 참고 사이트: [URL 또는 설명]

구체적으로:
1. 색상은 최대 3가지만 사용
2. 글꼴은 읽기 쉬운 것으로
3. 요소 사이에 여백을 충분히
4. 모바일에서도 잘 보이게 (반응형)
5. 버튼은 손가락으로 누르기 쉬운 크기로
```

### 템플릿 11. 레이아웃 변경하기 (Chapter 9)

```
내 웹페이지 레이아웃을 바꿔줘.

현재: [현재 구조 설명]
원하는 구조:
- 상단: [메뉴바/로고/검색창]
- 가운데: [메인 콘텐츠 설명]
- 하단: [푸터/연락처/링크]

[참고 사이트]처럼 만들어줘.
```

### 템플릿 12. 피드백 정리 요청하기 (Chapter 10)

```
내 서비스에 대한 사용자 피드백을 정리해줘.

아래 의견들을 3가지로 분류해줘:
- "바로 수정" (2명 이상 같은 말, 서비스 사용 불가)
- "나중에" (1명만 언급, 추가 기능 요청)
- "무시" (서비스 방향과 안 맞음)

피드백 목록:
1. [피드백 1]
2. [피드백 2]
3. [피드백 3]
4. [피드백 4]
5. [피드백 5]

각 피드백에 대해 분류 이유도 간단히 설명해줘.
```

## Part 4. 수익화

### 템플릿 13. 수익 모델 상담하기 (Chapter 11)

```
내 서비스는 [서비스 설명]이야.
타겟 사용자는 [대상]이고,
현재 [사용자 수/상황]이야.

이 서비스에 가장 적합한 수익 모델을 추천해줘.
각 모델별로 장단점과 예상 수익을 알려줘.

고려해줘:
- 1인으로 운영
- 초기 비용 최소화
- 한국 시장 기준
```

### 템플릿 14. 가격 설정 도움받기 (Chapter 11)

```
내 서비스의 가격을 정하려고 해.

서비스: [서비스 설명]
타겟: [대상]
경쟁 서비스: [비슷한 서비스 이름과 가격]

3단계 가격표를 만들어줘:
- 무료 플랜: 기본 기능
- 프로 플랜: 핵심 기능 전부
- 프리미엄 플랜: 모든 기능 + 추가 혜택

각 플랜의 가격과 포함 기능을 표로 정리해줘.
한국 시장 기준으로 적절한 가격대를 제안해줘.
```

### 템플릿 15. 결제 기능 추가하기 (Chapter 12)

```
내 웹사이트에 결제 기능을 추가해줘.

결제 수단: [Stripe / 토스페이먼츠]
상품: [상품명]
가격: [금액]원 ([월 구독 / 일회성])

결제 완료 후:
- [감사 페이지로 이동 / 이메일 발송 / 콘텐츠 접근 허용]

테스트 모드로 먼저 만들어줘.
```

### 템플릿 16. 마케팅 콘텐츠 작성하기 (Chapter 13)

```
내 서비스를 홍보하기 위한 [블로그 글 / SNS 게시물]을 써줘.

서비스: [서비스 설명]
타겟 독자: [대상]
채널: [네이버 블로그 / 인스타그램 / 트위터]

주제: [주제]

조건:
- [친근한/전문적인] 말투
- 서비스를 자연스럽게 녹여줘 (광고 느낌 X)
- SEO를 고려해서 제목에 검색 키워드 포함
- [500자/1000자] 내외로
```

### 템플릿 17. 프리미엄 기능 추가하기 (Chapter 14)

```
지금 서비스에 프리미엄 구독 기능을 추가해줘.

무료 사용자:
- [제한된 기능 1]
- [제한된 기능 2]
- 광고 있음

프리미엄 사용자 (월 [금액]원):
- [전체 기능 1]
- [전체 기능 2]
- 광고 없음
- [추가 혜택]

프리미엄 가입 유도 배너도 만들어줘.
```

## Part 5. 지속 가능한 성장

### 템플릿 18. 자동 이메일 설정하기 (Chapter 15)

```
새 사용자가 가입하면 자동 이메일을 보내줘.

이메일 시퀀스:
1. 가입 즉시: 환영 이메일 + 시작 가이드 링크
2. 3일 후: 주요 기능 소개 이메일
3. 7일 후: 사용 팁 + 프리미엄 안내

이메일 톤: 친근하고 따뜻하게
발신자 이름: [이름]
서비스 이름: [서비스명]
```

### 템플릿 19. 고객 응대 챗봇 만들기 (Chapter 15)

```
내 서비스에 고객 응대 챗봇을 추가해줘.

자주 묻는 질문과 답변:
1. Q: 비밀번호를 잊어버렸어요
   A: [답변]
2. Q: 결제는 어떻게 하나요?
   A: [답변]
3. Q: 환불이 가능한가요?
   A: [답변]
4. Q: 이 기능은 어떻게 쓰나요?
   A: [답변]
5. Q: [추가 질문]
   A: [답변]

챗봇이 모르는 질문은 내 이메일 [이메일]로 전달해줘.
```

### 템플릿 20. 서비스 모니터링 설정하기 (Chapter 15)

```
내 서비스가 정상 작동하는지 자동으로 확인해줘.

모니터링할 것:
1. 웹사이트 접속 가능 여부 (매 10분)
2. 결제 시스템 작동 여부
3. 에러 발생 시 즉시 알림
4. 매일 밤 12시 자동 백업

문제 발생 시:
- 내 이메일 [이메일]로 알림
- 최근 30일 백업 보관
```

### 템플릿 21. 커뮤니티 운영 가이드 요청하기 (Chapter 16)

```
내 서비스의 사용자 커뮤니티를 시작하려고 해.

서비스: [서비스 설명]
플랫폼: [카카오톡 오픈채팅 / 디스코드 / 네이버 카페]
예상 초기 인원: 5~10명

아래 내용을 만들어줘:
1. 커뮤니티 이름 제안 3가지
2. 커뮤니티 소개글
3. 운영 규칙 3가지
4. 첫 주 운영 계획 (매일 올릴 글 주제)
5. 주간 정기 이벤트 아이디어 3가지
```

### 템플릿 22. 12개월 사업 로드맵 세우기 (Chapter 17)

```
내 1인 사업의 12개월 로드맵을 만들어줘.

서비스: [서비스 설명]
현재 상태: [아이디어 단계 / MVP 완성 / 첫 수익 발생]
목표: [12개월 후 목표]

분기별로 나눠줘:
- 1~3개월: MVP 완성 목표
- 4~6개월: 첫 수익 목표
- 7~9개월: 성장 목표
- 10~12개월: 안정화 목표

각 분기마다:
- 핵심 할 일 3가지
- 수치 목표 (사용자 수, 수익)
- 주의할 점

현실적으로, 1인이 하루 2~3시간 투자하는 기준으로 세워줘.
```

## 보너스: 만능 수정 프롬프트

### 결과가 마음에 안 들 때

```
방금 결과에서 이 부분을 바꿔줘:
- [바꾸고 싶은 부분]을 [원하는 방향]으로

나머지는 그대로 유지해줘.
```

### 더 쉬운 설명이 필요할 때

```
방금 설명을 더 쉽게 다시 해줘.
전문 용어 없이, 초등학생도 이해할 수 있게.
비유를 들어서 설명해줘.
```

### 선택지가 필요할 때

```
[요청 사항]에 대해 3가지 다른 방향을 제안해줘.
각각의 장단점도 알려줘.
내가 선택할게.
```
