# 전자책 완성과 디자인

<div class="chapter-desc">편집, 교정, 표지 디자인, PDF 제작까지 — 원고를 판매 가능한 상품으로 만듭니다.</div>

## 편집과 교정

### 3단계 편집 프로세스

좋은 전자책은 **최소 3번의 편집**을 거칩니다.

| 단계 | 목적 | 방법 |
|------|------|------|
| 1차: 구조 편집 | 전체 흐름과 구조 확인 | 목차만 놓고 논리 흐름 점검 |
| 2차: 내용 편집 | 정보의 정확성, 깊이 확인 | 챕터별로 꼼꼼히 읽기 |
| 3차: 문장 교정 | 맞춤법, 문체, 가독성 | AI + 수동 교정 |

### Claude를 활용한 교정

<div class="box-prompt">

**교정 프롬프트:**

<pre>
다음 글의 맞춤법, 문법, 문체를 교정해줘.

[원고 텍스트]

교정 기준:
1. 한국어 맞춤법 규정 준수
2. 문장 길이 30자 이내 권장
3. 존댓말 통일 (합니다/해요 → 합니다체)
4. 수동태 → 능동태 변환
5. 불필요한 부사/형용사 제거

원본과 수정본을 나란히 보여주고,
변경 이유를 설명해줘.
</pre>

</div>

<div class="box-tip">

맞춤법 검사는 Claude 외에도 **네이버 맞춤법 검사기**와 **부산대 맞춤법 검사기**를 병행하세요. AI도 맞춤법을 틀릴 수 있습니다.

</div>

## 표지 디자인

### 표지가 중요한 이유

온라인에서 전자책의 첫인상은 **표지 썸네일**입니다. 3초 안에 클릭을 유도할 수 있어야 합니다.

### 좋은 표지의 4요소

1. **선명한 제목** — 썸네일 크기에서도 읽을 수 있어야 함
2. **깔끔한 컬러** — 2~3가지 색상만 사용
3. **적절한 이미지** — 주제를 직관적으로 전달
4. **프로페셔널한 느낌** — 아마추어 느낌은 신뢰를 떨어뜨림

### 무료/저비용 표지 제작 도구

| 도구 | 비용 | 난이도 | 특징 |
|------|------|--------|------|
| Canva | 무료/프로 | 쉬움 | 전자책 템플릿 풍부 |
| Figma | 무료 | 중간 | 자유도 높음 |
| 미리캔버스 | 무료 | 쉬움 | 한국어 템플릿 |
| AI 이미지 생성 | API 비용 | 쉬움 | 고유한 이미지 생성 |

## PDF 제작

### 권장 PDF 스펙

| 항목 | 설정값 |
|------|--------|
| 페이지 크기 | A4 (210 × 297mm) |
| 여백 | 상하 25mm, 좌우 22mm |
| 본문 폰트 | Pretendard 또는 Noto Sans KR |
| 본문 크기 | 11pt |
| 줄간격 | 1.7 |
| 코드 폰트 | D2Coding |
| 이미지 해상도 | 150 DPI 이상 |

### 마크다운 → PDF 변환 도구

<div class="box-key">

**이 가이드에서 사용하는 빌드 파이프라인:**

1. **마크다운** 원고 작성 → 2. **HTML** 변환 (markdown-it) → 3. **CSS** 스타일 적용 → 4. **PDF** 생성 (Puppeteer)

명령어: `npm run build`

</div>

<div class="box-warning">

PDF 생성 전 반드시 확인할 것:
- 이미지가 모두 정상 표시되는지
- 페이지 넘김이 자연스러운지 (표/박스가 중간에 잘리지 않는지)
- 하이퍼링크가 정상 작동하는지
- 폰트가 임베드되어 있는지 (다른 환경에서 깨지지 않도록)

</div>
