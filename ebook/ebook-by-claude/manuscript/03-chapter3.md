## 전자책의 기본 구조

잘 팔리는 전자책에는 공통된 구조가 있습니다. 독자가 처음부터 끝까지 자연스럽게 읽을 수 있는 흐름을 만들어야 합니다.

### 표준 전자책 구조

```
1. 표지
2. 저작권 / 면책 조항
3. 목차
4. 프롤로그 (왜 이 책을 썼는가)
5. 본문 챕터 (5~10개)
   - 도입 → 이론 → 실습 → 정리
6. 에필로그 (다음 단계 안내)
7. 부록 (템플릿, 체크리스트)
8. 저자 소개
```

### 적정 분량 가이드

| 전자책 유형 | 페이지 수 | 단어 수 (한글) | 가격대 |
|-------------|-----------|---------------|--------|
| 미니 가이드 | 20~40p | 8,000~15,000자 | 5,000~9,900원 |
| 표준 전자책 | 50~100p | 20,000~40,000자 | 9,900~19,900원 |
| 프리미엄 가이드 | 100~200p | 40,000~80,000자 | 19,900~39,900원 |
| 종합 핸드북 | 200p+ | 80,000자+ | 29,900원+ |

<div class="box-tip">

처음이라면 **50~80페이지 분량**을 목표로 하세요. 너무 짧으면 가치를 느끼기 어렵고, 너무 길면 완성하기 어렵습니다.

</div>

### 목차 검증 포인트

목차를 만들었다면 다음을 확인하세요:

1. **독자 여정이 자연스러운가?** — 모르는 상태 → 아는 상태 → 할 수 있는 상태
2. **각 챕터가 독립적으로도 가치 있는가?** — 하나만 읽어도 배움이 있어야 함
3. **실습이 포함되어 있는가?** — 읽기만 하는 책은 가치가 떨어짐
4. **분량이 균형 잡혀 있는가?** — 한 챕터가 너무 길거나 짧지 않게

## 챕터 구조 템플릿

각 챕터는 다음 구조를 따릅니다:

```
## [챕터 제목]

### 도입 (왜 이것이 중요한가)
- 독자의 현재 문제점 제시
- 이 챕터를 읽으면 얻는 것

### 핵심 내용 (어떻게 하는가)
- 이론과 방법론
- 단계별 프로세스
- 구체적 예시

### 실전 적용 (직접 해보기)
- 실습 과제
- 템플릿 / 워크시트

### 정리
- 핵심 포인트 3가지
- 다음 챕터 연결
```
