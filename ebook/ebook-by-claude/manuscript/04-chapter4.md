# Claude로 본문 작성하기

<div class="chapter-desc">프롬프트 전략부터 실전 작성법, 퇴고까지 AI와 함께 글쓰기의 모든 것.</div>

## 프롬프트 엔지니어링 기초

Claude와 효과적으로 작업하려면 **좋은 지시(프롬프트)**를 줄 수 있어야 합니다. 프롬프트의 품질이 곧 결과물의 품질입니다.

### 좋은 프롬프트의 5요소

| 요소 | 설명 | 예시 |
|------|------|------|
| **역할(Role)** | Claude에게 부여할 전문가 역할 | "너는 10년 경력의 재테크 전문가야" |
| **맥락(Context)** | 배경 정보와 상황 | "초보 투자자를 위한 전자책을 쓰고 있어" |
| **지시(Task)** | 구체적인 작업 내용 | "Chapter 3의 '분산 투자' 섹션을 써줘" |
| **형식(Format)** | 출력 형태 | "마크다운 형식, 2000자 내외, 예시 포함" |
| **제약(Constraint)** | 주의 사항 | "전문 용어는 쉽게 풀어 설명해줘" |

### 마스터 프롬프트 (시스템 프롬프트)

<div class="box-prompt">

**전자책 작성용 마스터 프롬프트:**

<pre>
너는 전자책 전문 작가야. 다음 규칙을 따라 글을 써줘:

[전자책 정보]
- 제목: [제목]
- 대상 독자: [타겟 독자]
- 톤: 친근하지만 전문적, 존댓말 사용

[작성 규칙]
1. 문장은 짧고 명확하게 (한 문장 30자 이내 권장)
2. 한 단락은 3~5문장으로 구성
3. 추상적 설명 대신 구체적 예시를 사용
4. 전문 용어에는 반드시 쉬운 설명을 덧붙여
5. 각 섹션에 실행 가능한 조언을 포함

[형식]
- 마크다운 문법 사용
- h2(##)로 대주제, h3(###)로 소주제
- 핵심 내용은 표 또는 리스트로 정리
- 적절한 위치에 요약 박스 삽입
</pre>

</div>

## 챕터별 작성 전략

### 단계 1: 초안 생성

하나의 챕터를 한 번에 쓰려 하지 마세요. **섹션 단위**로 나눠서 작성하면 품질이 훨씬 좋아집니다.

<div class="box-prompt">

**섹션 작성 프롬프트:**

<pre>
[마스터 프롬프트 적용 상태에서]

Chapter [번호]의 "[섹션 제목]" 섹션을 써줘.

이 섹션의 목표:
- [이 섹션에서 독자가 배울 내용]

포함할 내용:
- [핵심 개념 1]
- [핵심 개념 2]
- [실제 사례 1개]

분량: 약 [1000]자
</pre>

</div>

### 단계 2: 퇴고와 개선

초안을 받았다면 바로 사용하지 마세요. 다음 프로세스를 거쳐야 합니다:

1. **1차 검토**: Claude에게 자체 검토 요청
2. **개인화**: 여러분의 경험과 사례 추가
3. **팩트체크**: 통계, 수치, 고유명사 확인
4. **톤 통일**: 전체 챕터의 문체가 일관되게 조정

<div class="box-prompt">

**퇴고 프롬프트:**

<pre>
다음 글을 검토하고 개선해줘:

[작성한 초안 붙여넣기]

검토 기준:
1. 문장이 너무 길거나 복잡한 곳은 없는지
2. 추상적이고 모호한 표현은 없는지
3. 중복되는 내용은 없는지
4. 논리적 흐름이 자연스러운지
5. 독자가 실제로 실행할 수 있는 구체적 조언이 있는지

개선된 버전을 작성하고, 변경 사항을 목록으로 알려줘.
</pre>

</div>

### 단계 3: 인간적 요소 추가

<div class="box-warning">

AI가 생성한 글은 **정확하지만 무미건조할 수 있습니다**. 반드시 다음을 직접 추가하세요:

- 개인 경험담 ("제가 처음 전자책을 썼을 때...")
- 실패 사례 (실패에서 배운 교훈)
- 감정적 공감 ("이 과정이 막막하게 느껴질 수 있어요")
- 독자와의 대화 ("여러분은 어떠세요?")

</div>

## 글쓰기 품질 체크리스트

| 체크 항목 | 확인 |
|-----------|------|
| 모든 문장이 30자 이내인가? | ☐ |
| 전문 용어에 설명이 있는가? | ☐ |
| 각 섹션에 구체적 예시가 있는가? | ☐ |
| 개인 경험이 최소 1개 포함되어 있는가? | ☐ |
| 독자가 실행할 수 있는 액션이 있는가? | ☐ |
| 이전/다음 섹션과 흐름이 자연스러운가? | ☐ |

<div class="box-tip">

하루에 한 챕터씩 작성하세요. 한 번에 모든 챕터를 쓰면 품질이 떨어집니다. **"오늘은 Chapter 3만"**이라는 마인드가 중요합니다.

</div>
